Task Queue Orchestrator
This visualization demonstrates how an orchestrator manages tasks with specific rules:
  • If there are duplicate entries in the queue, the oldest one is deleted
  • If a task is in progress and a duplicate appears, it's kept pending until the current task completes
  • At maximum, only 3 tasks can run in parallel
